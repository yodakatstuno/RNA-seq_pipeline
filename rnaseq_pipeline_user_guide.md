この文書は、このリポジトリに実装されている RNA-seq パイプラインの解析手法を、エンドユーザー向けに説明するものです。RNA-seq の基本は知っているが、統計やプログラミングにはあまり慣れていない利用者でも理解できるように書いています。

## 1. 概要

### このパイプラインでできること

このパイプラインは、RNA-seq の生カウント行列とサンプルメタデータを入力として受け取り、以下のような解析を段階的に実行できます。

- 差次的発現解析
- 正規化
- クラスタリング
- 時系列解析
- 次元削減
- 機能アノテーション・経路解析
- 品質管理
- 回帰解析
- 表現型関連解析
- 機械学習
- 共発現・ネットワーク解析
- Seurat を用いた単一細胞風の解析

このパイプラインは複数生物種に対応しており、ゼブラフィッシュも対象に含まれます。

### 典型的なワークフロー

よくある利用手順は次のとおりです。

1. 生のカウント行列とメタデータ表を準備する。
2. カウント行列の列名とメタデータのサンプル ID が一致しているか確認する。
3. 正規化または変換を行う。
4. PCA、UMAP、クラスタリング、QC でサンプル全体の構造を確認する。
5. 差次的発現解析を行う。
6. 時間依存の実験であれば、時系列クラスタリングを行う。
7. GO、KEGG、Reactome、GSEA などで生物学的意味を解釈する。
8. 必要に応じて、回帰解析、表現型関連解析、機械学習、ネットワーク解析に進む。

### 全体像のイメージ

```text
生カウント
   |
   v
サンプル対応付け / QC
   |
   v
正規化 / 変換
   |
   +--> 差次的発現解析
   +--> PCA / UMAP / クラスタリング
   +--> 時系列クラスタリング
   +--> 機能アノテーション / GSEA
   +--> 回帰 / 表現型 / 機械学習 / ネットワーク
```

## 2. データ入力と変数

### カウント行列

カウント行列は、このパイプラインの主要な数値入力です。

- 行 = 遺伝子
- 列 = サンプル
- 値 = 生のリードカウント

例:

| gene | sample_1 | sample_2 | sample_3 |
|---|---:|---:|---:|
| geneA | 120 | 150 | 98 |
| geneB | 5 | 2 | 11 |

### カウントの生物学的意味

カウントは、ある遺伝子に割り当てられたシーケンスリード数です。一般にカウントが大きいほど発現量が高いと考えられますが、生カウントはシーケンス深度やライブラリ構成の偏りの影響を受けるため、そのままではサンプル間比較に使えません。

### カウントの数理的意味

遺伝子 \(g\)、サンプル \(i\) に対する生カウントを \(Y_{gi}\) とすると、

\[
Y_{gi} \in \{0,1,2,\dots\}
\]

です。これは連続量ではなく整数のカウントデータです。そのため、DESeq2 や edgeR のような手法では、正規分布ではなくカウントデータ向けの確率モデルで扱います。

### メタデータ表

メタデータ表は、各サンプルの属性を記述する表です。

典型的な列は以下です。

- `sample_id`
- `condition`
- `genotype`
- `dpf`
- `batch`

### メタデータ変数の生物学的意味と数理的意味

| 変数 | 生物学的意味 | パイプライン内での数理的意味 |
|---|---|---|
| `sample_id` | 生物学的サンプルの識別子 | 識別子。カウント行列とメタデータの対応付けに使う |
| `condition` | 対照・処理群などの主たる実験条件 | 群比較に使うカテゴリ変数 |
| `genotype` | WT、Het、Ho、MUT などの遺伝的背景 | カテゴリ変数 |
| `dpf` | days post fertilization などの時間・発生段階 | DE では因子として使われることが多く、時系列クラスタリングでは時間軸として使われる |
| `batch` | 技術的処理バッチ | バッチ共変量 |

### 解釈上の重要点

- **因子 factor**
  名前付きのカテゴリ変数です。例: WT、Ho、MUT。
- **共変量 covariate**
  変動の説明に使う変数です。例: batch、dpf、表現型スコア。
- **グループ化変数 grouping variable**
  サンプルを色分けしたり群分けしたりするための変数です。例: condition、genotype。

### このパイプラインでの実際の使われ方

- `sample_id` は、カウント行列の列とメタデータの行を対応付けるために使われます。
- `condition` は、差次的発現解析で既定の主比較変数として使われます。
- `genotype` と `dpf` は、交互作用モデルや VST の設計式で使われます。
- `dpf` は、時系列クラスタリングで時間軸として自動検出されます。
- `batch` は、バッチ補正で使われます。

## 3. 正規化手法

このパイプラインは `R/04_normalization.R` で以下の手法を実装しています。

- DESeq2 size factor normalization
- TMM
- CPM
- log2(count + 1)
- VST
- rlog
- Z-score

重要な点:

- Z-score は、概念的には**正規化法そのものではありません**。
- パイプラインでは `scaled_zscore.*` として保存できますが、これは下流変換後の行列です。
- 時系列クラスタリングモジュールでは、Z-score を外部ファイルに依存せず、内部で毎回再計算します。

### なぜ正規化が必要なのか

RNA-seq サンプル間には、しばしば次のような違いがあります。

- シーケンス深度が異なる
- ライブラリ構成が異なる
- 平均と分散の関係が異なる

これらを補正または緩和しないと、見かけ上の違いが生物学的差ではなく技術的差になってしまいます。

---

### 3.1 DESeq2 size factor normalization

#### A. 何をする方法か

各サンプルをスケーリングし、サンプル間でカウントを比較しやすくします。

#### B. なぜ必要か

あるサンプルの総リード数が別のサンプルの 2 倍なら、生物学的差がなくても生カウントは大きく見えます。その影響を減らすためです。

#### C. 数学的な考え方

DESeq2 は各サンプル \(i\) に対して size factor \(s_i\) を推定します。

\[
\text{normalized count}_{gi} = \frac{Y_{gi}}{s_i}
\]

size factor は単純な総リード数割りではなく、median-ratio に基づく頑健な推定です。

#### D. いつ使うべきか

- DESeq2 ベースの解析の基礎正規化として
- カウントデータを意識した解析で

#### E. いつ向かないか

- PCA やクラスタリング用に分散安定化済みの値が欲しいとき
- 遺伝子ごとのパターン比較のために標準化された値が欲しいとき

---

### 3.2 TMM (Trimmed Mean of M-values, edgeR)

#### A. 何をする方法か

TMM はサンプル間の組成バイアスを減らすようにスケーリングします。

#### B. なぜ必要か

一部の遺伝子が非常に高発現だと、単純なライブラリサイズ補正では不十分なことがあります。TMM はその影響を抑えます。

#### C. 数学的な考え方

TMM は、各サンプルを参照サンプルと比較し、遺伝子ごとの対数比を計算し、極端な遺伝子をトリミングした上でスケーリング係数を求めます。

直感的には、

\[
\text{adjusted count} \approx \frac{Y_{gi}}{\text{effective library size}_i}
\]

です。

このパイプラインでは、その後 `edgeR::cpm(..., normalized.lib.sizes=TRUE)` を使って正規化値を出力します。

#### D. いつ使うべきか

- edgeR 系のスケーリングを使いたいとき
- ライブラリ構成バイアスが気になるとき

#### E. いつ向かないか

- DESeq2 流の正規化を一貫して使いたいとき
- 探索的クラスタリング用に分散安定化行列が欲しいとき

---

### 3.3 CPM (Counts Per Million)

#### A. 何をする方法か

カウントをライブラリサイズで割り、「100 万リードあたり」の値として表します。

#### B. なぜ必要か

サンプル間比較がしやすくなり、可視化にも使いやすい単位になります。

#### C. 数学的な考え方

基本形は、

\[
\text{CPM}_{gi} = \frac{Y_{gi}}{\text{library size}_i} \times 10^6
\]

です。

このパイプラインでは、CPM は edgeR 正規化後に出力されるため、分母は生のライブラリサイズではなく調整後の effective library size になります。

#### D. いつ使うべきか

- 一般ユーザー向けにわかりやすい発現量指標が欲しいとき
- 可視化をしたいとき

#### E. いつ向かないか

- DESeq2 による厳密な差次的発現モデルの代用として
- 平均依存の分散を安定化したいとき

---

### 3.4 log2(count + 1)

#### A. 何をする方法か

生カウントに単純な対数変換をかけます。

#### B. なぜ必要か

生カウントは右に長く裾を引く分布になりやすく、そのままでは図が見づらくなります。対数変換により、大きな値の圧縮が行われます。

#### C. 数学的な考え方

\[
\log_2(Y_{gi} + 1)
\]

`+1` は、0 の対数を避けるためです。

#### D. いつ使うべきか

- 手早い探索図を作りたいとき
- 軽量な変換で十分なとき
- 直感的で人に説明しやすい変換が欲しいとき

#### E. いつ向かないか

- 低カウントのノイズが大きいとき
- より統計的に強い分散安定化変換が望ましいとき

---

### 3.5 VST (Variance Stabilizing Transformation)

#### A. 何をする方法か

低発現遺伝子と高発現遺伝子で分散が極端に違いすぎないように、カウントを変換します。

#### B. なぜ必要か

RNA-seq カウントには平均と分散の関係があります。

- 低カウントはノイズが大きい
- 高カウントは比較的安定している

この性質は PCA、クラスタリング、ヒートマップをゆがめることがあります。VST はその問題を緩和します。

#### C. 数学的な考え方

変換式そのものはデータから学習されます。直感的には、

\[
\text{Var}(\text{transformed values}) \approx \text{constant}
\]

となるように、平均に応じて適切に変換することを目指します。

このパイプラインでは、

- `DESeq2::vst(..., blind=FALSE)` を主手法として使います
- `genotype` と `dpf` の両方が存在すれば設計式は `~ genotype + dpf`
- それ以外は `~ 1`

です。

また、小さい行列に対して `vst()` が内部の `nsub` 条件で失敗した場合は、警告をログに出した上で、

- `DESeq2::varianceStabilizingTransformation(..., blind=FALSE)`

へフォールバックします。

#### D. いつ使うべきか

- 探索解析の既定変換として
- PCA
- クラスタリング
- 時系列クラスタリング
- ヒートマップ

#### E. いつ向かないか

- 差次的発現検定そのものに使う生カウントモデルが必要なとき
- 学習型変換を避け、もっと単純な変換で済ませたいとき

---

### 3.6 rlog (Regularized log)

#### A. 何をする方法か

rlog は DESeq2 の別の変換法で、低カウントの不安定さを縮小しつつ、安定した log 風の発現行列を作ります。

#### B. なぜ必要か

特にサンプル数が少ないデータでは、低カウントの揺らぎが大きく、rlog が有効なことがあります。

#### C. 数学的な考え方

rlog は単純な対数変換に似ていますが、低カウントの不安定な観測値に正則化をかけます。

イメージとしては、

\[
\text{rlog}(Y_{gi}) \approx \text{縮小を伴う log 風変換}
\]

です。

厳密な閉形式で表すよりも重要なのは、「ノイズの大きい低カウントをそのまま使わず、より安定な値に寄せる」という考え方です。

このパイプラインでは、

- `DESeq2::rlog(..., blind=FALSE)`
- ただしサンプル数が大きすぎる場合は既定では避ける
- `--force_rlog TRUE` を指定すると大きめのデータでも許可できる

という挙動です。

#### D. いつ使うべきか

- 小規模から中規模のサンプル数
- 探索的解析
- rlog の計算コストが問題にならない PCA やクラスタリング

#### E. いつ向かないか

- 大規模データ
- VST で十分なとき

---

### 3.7 Z-score

#### A. 何をする方法か

各遺伝子について、サンプル間の平均を 0、標準偏差を 1 にそろえます。

#### B. なぜ必要か

絶対量の違いを消し、発現変化の**形**を強調するためです。

たとえば、2 つの遺伝子が全く違う発現量スケールを持っていても、時間に対する上がり方・下がり方が同じなら、Z-score によって同じパターンとして比較しやすくなります。

#### C. 数学的な考え方

各遺伝子ごとに、

\[
Z = \frac{x - \mu}{\sigma}
\]

を計算します。

ここで、

- \(x\) = あるサンプルでの発現値
- \(\mu\) = その遺伝子のサンプル間平均
- \(\sigma\) = その遺伝子のサンプル間標準偏差

です。

#### D. いつ使うべきか

- ヒートマップ
- 軌跡クラスタリング
- 発現量そのものではなく変化パターンを比べたいとき

#### E. いつ向かないか

- 正規化の代用として
- 絶対発現量そのものが生物学的に重要なとき

## 4. 差次的発現解析 (DESeq2)

このパイプラインの差次的発現解析は `R/01_differential_expression.R` で実装されており、以下を扱えます。

- DESeq2
- edgeR
- limma

このうち、特に時間と遺伝子型が関わる実験で統計的に重要なのが**交互作用モデル**です。

### モデル

ユーザーが

- `design_mode=interaction`
- `factors=dpf,genotype`

を選ぶと、パイプラインは次の設計式を構築します。

\[
\text{expression} \sim \text{dpf} + \text{genotype} + \text{dpf}:\text{genotype}
\]

これは次と同じ意味です。

\[
\text{gene expression} \sim \text{genotype} + \text{time} + \text{genotype} \times \text{time}
\]

### 線形モデルとは何か

線形モデルは、観測された応答を複数の効果の和として説明しようとする枠組みです。

直感的には、

```text
観測された発現
= 基準値
+ 遺伝子型の効果
+ 時間の効果
+ 交互作用の効果
+ ランダムなばらつき
```

です。

### factor とは何か

factor はカテゴリ変数です。

例:

- genotype = WT, Ho
- dpf = 3, 5, 7
- condition = control, treatment

このパイプラインでは、DE に使う要因はモデル化の際に因子として扱われます。

### 交互作用項とは何か

交互作用項は、次の問いを表します。

> 時間の効果は、遺伝子型によって変わるか。

交互作用がないモデルでは、

- すべての遺伝子型は同じ時間変化をし、ただ上下にずれているだけ

とみなされます。

交互作用があるモデルでは、

- 遺伝子型ごとに時間変化の軌跡そのものが異なってよい

と考えます。

### 主効果と交互作用効果

\[
\text{expression} \sim \text{genotype} + \text{dpf} + \text{genotype}:\text{dpf}
\]

において、

- **genotype の主効果**: 遺伝子型間の平均差
- **dpf の主効果**: 時間に沿った平均的変化
- **交互作用効果**: 時間変化の形が遺伝子型によって異なるか

を意味します。

### 生物学的に重要な理由

たとえば WT と Ho が 3 dpf ではよく似ているが、7 dpf では大きく離れるとします。

これは単なる遺伝子型差ではありません。
**遺伝子型 × 時間の交互作用**です。

### LRT (Likelihood Ratio Test)

このパイプラインでは、

- `Wald` 検定
- `LRT` 検定

を選べます。

LRT の考え方は次のとおりです。

1. **完全モデル full model** を当てはめる
2. **縮約モデル reduced model** を当てはめる
3. 完全モデルが有意に良い説明を与えるかを調べる

数式では、

\[
\text{LRT statistic} = 2(\log L_{\text{full}} - \log L_{\text{reduced}})
\]

です。ここで \(L\) は尤度です。

### LRT の直感

もし完全モデルに交互作用項が入り、縮約モデルに入らないなら、LRT は次を問います。

> genotype × time の項を加えることで、データ説明は有意に改善するか。

生物学的には、

> 時間変化のふるまいは遺伝子型によって異なるか。

を検定していると考えればよいです。

### よくある比較

完全モデル:

\[
\sim \text{dpf} + \text{genotype} + \text{dpf}:\text{genotype}
\]

縮約モデル:

\[
\sim \text{dpf} + \text{genotype}
\]

完全モデルの方が大きく良ければ、交互作用は重要です。

## 5. 時系列解析の考え方

### 時間依存的な遺伝子発現とは

遺伝子によっては、発生時間、処理時間、病態進行段階に応じて発現が変化します。

例:

- gene A は 3 dpf から 7 dpf にかけて単調に増加する
- gene B は 5 dpf でピークを持つ
- gene C は変異体でのみ低下する

### trajectory とは何か

trajectory は、順序づけられたサンプルに沿った遺伝子の発現パターンです。

例:

```text
Gene X: low -> medium -> high
Gene Y: high -> medium -> low
Gene Z: low -> high -> low
```

### なぜクラスタリングが有用か

時系列 RNA-seq では、何千もの遺伝子が時間とともに変化します。クラスタリングにより、似た軌跡を持つ遺伝子をまとめられます。

これによって、

- 初期応答遺伝子
- 後期応答遺伝子
- 一過性ピーク遺伝子
- 遺伝子型特異的パターン

を見つけやすくなります。

専用の時系列クラスタリングモジュールは `R/05_clustering.R` です。

既定の流れは、

1. VST で変換する
2. 遺伝子を選ぶ
3. 遺伝子ごとに Z-score を再計算する
4. 遺伝子をクラスタリングする
5. クラスター平均軌跡を可視化する

です。

## 6. クラスタリング手法

このパイプラインでは、クラスタリングは主に 2 つの場所で実装されています。

- `R/02_sample_clustering.R`: サンプル中心のクラスタリング、および旧来の遺伝子軌跡表示
- `R/05_clustering.R`: 専用の時系列遺伝子クラスタリング

必須のクラスタリング法は、

- hierarchical clustering
- k-means clustering

です。

### 6.1 階層的クラスタリング

#### A. 直感

階層的クラスタリングは、最も似ているもの同士を順に結合していき、最終的に木構造を作ります。

出力は dendrogram です。

#### B. 数学的な考え方

まず 2 つの遺伝子、または 2 つのサンプルの距離を定義します。

ユークリッド距離なら、

\[
d(x,y)=\sqrt{\sum_j (x_j-y_j)^2}
\]

です。

その後、linkage rule に従って近いものから順に結合していきます。

このパイプラインでは、

- Euclidean
- Pearson
- Spearman

の距離を使えます。また、多くのクラスタリング図では `ward.D2` が使われます。

#### C. 長所

- 最初にクラスター数を決めなくてよい
- 樹形図が得られる
- 探索的解析に向く

#### D. 短所

- 距離の定義に結果が影響される
- 一度結合したものを後で最適化しにくい

#### E. 使いどころ

- 探索的クラスタリング
- ヒートマップ
- 木構造そのものに生物学的意味がありそうなとき

---

### 6.2 k-means クラスタリング

#### A. 直感

k-means は、遺伝子を \(k\) 個のグループに分け、同じグループ内では発現プロファイルが似るようにします。

#### B. 数学的な考え方

各クラスターは centroid を持ちます。アルゴリズムは次を繰り返します。

1. 各遺伝子を最も近い centroid に割り当てる
2. 各 centroid を再計算する

目的は、

\[
\sum_{k=1}^K \sum_{i \in C_k} \|x_i-\mu_k\|^2
\]

を小さくすることです。

ここで、

- \(C_k\) = クラスター \(k\)
- \(\mu_k\) = クラスター \(k\) の centroid

です。

時系列クラスタリングモジュールでは、k-means の入力行列は遺伝子ごとに Z-score 化されたものです。

#### C. 長所

- 単純
- 高速
- 明示的なクラスター割り当てが得られる

#### D. 短所

- \(k\) を自分で決める必要がある
- 初期値に影響される
- だいたい球状でコンパクトなクラスターを想定する

#### E. 使いどころ

- 軌跡クラスター数を決めて解析したいとき
- クラスター平均プロファイルを見たいとき

#### 再現性に関する注意

`R/05_clustering.R` では、k-means の前に

\[
\texttt{set.seed(123)}
\]

を設定しているため、結果は再現可能です。

## 7. Z-score 変換

これは重要なので独立した節として説明します。

### 式

\[
Z = \frac{x - \mu}{sd}
\]

### 意味

- \(Z=0\): その遺伝子にとって平均的な発現
- \(Z>0\): その遺伝子自身の平均より高い
- \(Z<0\): その遺伝子自身の平均より低い

### なぜ役立つか

たとえば、

- gene A は 10 から 20 の範囲
- gene B は 1000 から 2000 の範囲

だとしても、どちらも

```text
low -> medium -> high
```

という同じパターンを示しているかもしれません。

Z-score 化すると、発現量の大きさではなく**変化の形**を比べられます。

### 重要な警告

Z-score は**ライブラリサイズ補正でもカウント正規化でもありません**。

シーケンス深度の違いを補正するのではなく、すでに変換された値を標準化するだけです。

### このパイプラインでの実際の挙動

- `R/04_normalization.R` は `scaled_zscore.*` を保存できます
- `R/05_clustering.R` はその保存済みファイルに依存しません
- `R/05_clustering.R` は毎回内部で Z-score を再計算します

これは正しい設計です。クラスタリングでは、実際に今から使う行列をその場で標準化する必要があるからです。

## 8. 遺伝子選択 (`top_n`, `top_method`)

時系列クラスタリングモジュールでは、クラスタリング前に遺伝子数を絞れます。

これは重要です。すべての遺伝子をそのままクラスタリングすると、ノイズが多くなり解釈しにくくなることがよくあります。

### 8.1 分散ベースの選択

#### 直感

サンプル間で最も大きく変化する遺伝子を選びます。

#### 数学

遺伝子 \(g\) に対して、

\[
\mathrm{Var}(g)=\frac{1}{n-1}\sum_i (x_{gi}-\bar{x}_g)^2
\]

を計算します。

分散が大きいほど、その遺伝子はダイナミックに変化していると考えられます。

#### いつ有用か

- 探索的クラスタリング
- 時系列データ
- 軌跡が豊かな遺伝子を見つけたいとき

---

### 8.2 平均発現ベースの選択

#### 直感

平均発現量が高い遺伝子を選びます。

#### 数学

\[
\mathrm{Mean}(g)=\frac{1}{n}\sum_i x_{gi}
\]

です。

#### いつ有用か

- 十分に発現している遺伝子だけを見たいとき
- 極低発現遺伝子のノイズを避けたいとき

#### 限界

高発現だからといって、必ずしも最も動的な遺伝子とは限りません。

---

### 8.3 DEG ベースの選択

#### 直感

すでに差次的発現解析で「面白い」と判断された遺伝子だけを使ってクラスタリングします。

#### パイプラインでの挙動

パイプラインは `target_genes` ファイルを受け取り、通常は DE 結果由来の遺伝子一覧を想定しています。

利用する列としては、

- `gene`
- `significant`
- `padj`
- `pvalue`

などを参照します。

#### 数学

これは分散のような数式で順位付けする方法ではありません。

```text
まず差次的発現の証拠がある遺伝子を見つける
次にその遺伝子群だけをクラスタリングする
```

という、生物学的関心に基づくフィルタリングです。

#### いつ有用か

- 生物学的に意味のありそうな遺伝子に絞ってクラスタリングしたいとき
- データセットが大きく、より解釈しやすい部分集合が欲しいとき

#### 重要な警告

`top_method=deg` を指定しても、有効な遺伝子リストが与えられない場合、パイプラインは警告を出した上で variance ベース選択にフォールバックします。

## 9. 可視化

可視化は複数のステップで実装されています。主に以下です。

- `R/03_dimension_reduction.R`
- `R/04_normalization.R`
- `R/05_clustering.R`
- `R/10_sample_qc.R`
- `R/11_visualization.R`

### ヒートマップ

ヒートマップは、行列の値を色で表現した図です。

RNA-seq では主に次を見ます。

- サンプルが condition や batch ごとにまとまるか
- 遺伝子が共通パターンを持つブロックを作るか
- 時間順序が生物学的に自然に見えるか

行方向にスケーリングされたヒートマップでは、色は各遺伝子にとっての相対的な高低を表します。絶対発現量そのものではありません。

### クラスタープロファイル図

時系列クラスタリングモジュールでは、各クラスターの平均発現を描画します。

見るべき点は以下です。

- 単調増加または単調減少
- 初期ピーク
- 後期ピーク
- genotype 特異的な分岐
- 時間に沿って群が明確に分かれるか

### PCA / UMAP / t-SNE

これらは多数の遺伝子情報を少数の人工的な軸に圧縮し、サンプル構造を見やすくします。

見るべき点:

- condition による分離
- genotype ごとのまとまり
- dpf に沿った連続的変化
- batch による望ましくない分離

### Volcano plot

見るべき点:

- 大きな \(|\log_2 \text{fold change}|\)
- 小さい p 値
- 強い候補遺伝子がどれくらいあるか

### MA plot

見るべき点:

- fold change が平均発現に強く依存していないか
- 多くの遺伝子が 0 から離れているか

### QC plot

確認すべき点:

- ライブラリサイズの偏り
- 検出遺伝子数
- 発現分布
- サンプル間相関
- 外れ値候補

## 10. 実用的な使い方ガイド

### どの正規化法を選べばよいか

#### 探索的解析のため

使う候補:

- VST

理由:

- 分散が安定しやすい
- PCA に向く
- クラスタリングに向く
- ヒートマップに向く

#### 差次的発現検定のため

使うべきもの:

- 生カウントを用いた DESeq2 本来のモデル

理由:

- DESeq2 は生カウントを直接モデル化する
- 変換済み行列は探索や可視化のためであり、主検定そのものではない

#### クラスタリングのため

推奨:

- VST + 内部 Z-score

理由:

- VST はカウント依存の分散を抑える
- Z-score は軌跡の形に焦点を当てる

#### 軽いクイックチェックのため

使う候補:

- log2(count + 1)

#### edgeR 系スケーリングやわかりやすい発現単位が欲しいとき

使う候補:

- TMM
- CPM

### 目的別の推奨ワークフロー

#### 標準的な DE 実験

1. QC
2. 差次的発現解析
3. Volcano / heatmap / PCA
4. Enrichment または GSEA

#### genotype × time 実験

1. QC
2. 交互作用モデルで DE
3. VST
4. 時系列クラスタリング
5. クラスター軌跡の解釈
6. クラスター特異的遺伝子や DE 遺伝子に対する enrichment

#### 初回デバッグまたは学習用

サンプルデータモードを使います。

- `--example`
- `--sampledata`
- `--debug`

## 11. よくある落とし穴

### 落とし穴 1: 生カウントでそのままクラスタリングする

問題点:

- 生カウントはシーケンス深度と平均分散関係に強く支配される

よりよい方法:

- VST を行い、その後クラスタリング用に遺伝子ごとに Z-score 化する

### 落とし穴 2: Z-score を正規化法だと思ってしまう

なぜ誤りか:

- Z-score は遺伝子プロファイルを標準化するだけ
- ライブラリサイズやカウントスケールは補正しない

### 落とし穴 3: 交互作用項を無視する

genotype × time 実験で単純な 2 群比較だけをすると、本当に重要な生物学を見落とすことがあります。

問いが

> 遺伝子型によって時間変化の仕方が異なるか

なら、交互作用モデルが正しい枠組みです。

### 落とし穴 4: 情報の少ない遺伝子を入れすぎる

平坦な遺伝子やノイズ遺伝子が多いと、クラスタリングは解釈しにくくなります。

使うべき設定:

- `top_n`
- `top_method`

### 落とし穴 5: 絶対発現量とパターン形状を混同する

2 つの遺伝子は、同じ軌跡形状を持っていても発現量は大きく違うかもしれません。

使い分け:

- 発現量の大きさが重要なら raw scale に近い方法
- 形が重要なら Z-score

### 落とし穴 6: enrichment 解析を誤読する

ORA と GSEA は別の問いに答えます。

- ORA は、有意遺伝子が経路に過剰に含まれるかを問う
- GSEA は、経路全体が順位付き遺伝子リストの上位または下位に偏るかを問う

### 落とし穴 7: バッチ効果を無視する

PCA で biology ではなく batch によって分かれているなら、バッチ補正を行い、補正前後で PCA を比較すべきです。

## 12. コードと数理の対応関係

この節では、実際の入力項目が統計的にどう解釈されるかを対応付けます。

| 入力 | 数理的意味 | 使われる場所 |
|---|---|---|
| counts | 観測カウント応答 \(Y_{gi}\) | DE、正規化、QC、クラスタリング |
| `sample_id` | 添字・識別変数 | サンプル対応付け |
| `condition` | カテゴリ因子 | DE、可視化、表現型関連、機械学習 |
| `genotype` | カテゴリ因子 | 交互作用 DE、VST 設計、クラスタリングのラベル |
| `dpf` | 順序を持つ生物学的段階または時間共変量 | 交互作用 DE、VST 設計、時系列クラスタリング |
| `batch` | 迷惑共変量 nuisance covariate | バッチ補正 |
| `top_n` | 選択遺伝子数 | クラスタリング、可視化、ネットワーク解析 |
| `top_method` | 遺伝子選択ルール | 時系列クラスタリング |
| distance metric | 類似度の定義 | 階層的クラスタリング |
| \(k\) | クラスター数 | k-means |

### 具体例

メタデータが次のような場合を考えます。

| sample_id | genotype | dpf | condition |
|---|---|---:|---|
| s1 | WT | 3 | control |
| s2 | WT | 5 | control |
| s3 | Ho | 3 | treatment |
| s4 | Ho | 5 | treatment |

このとき、

- `genotype` は因子
- `dpf` は時間変数
- `condition` はグループ化変数
- counts は観測応答

です。

交互作用モデル

\[
\text{expression} \sim \text{genotype} + \text{dpf} + \text{genotype}:\text{dpf}
\]

は、

> 発現と時間の関係が genotype によって異なるか

を問います。

## 14. 最後に覚えておくべきこと

少数の原則だけ覚えるなら、次を覚えてください。

1. カウントベースの DE モデルには生カウントを使う。
2. 探索解析と時系列クラスタリングには VST を使う。
3. Z-score は遺伝子パターン形状を比べるためのものであり、カウント正規化ではない。
4. 生物学が genotype と time の両方に依存するなら、交互作用モデルを使う。
5. クラスタリングに入る遺伝子は `top_method` で制御する。
6. 生物学的解釈に入る前に、必ず QC と PCA を確認する。
